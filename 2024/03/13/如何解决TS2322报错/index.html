问题出在函数返回的泛型 T.

随后我将泛型 T 改为 { [key: string]: unknown } , 报错消失了.

a different subtype 什么意思?

  • A 是 S 的子类型: 类型 A 在类型 S 的基础上增加了额外属性.
  • A 和 B 是 S 的不同子类型: 类型 A 与类型 B 分别在类型 S 的基础上增加了不同的额外属性.

当你在 ts 中使用 type 关键字时, 例如: type A = { foo: 'Bar' }, 那么 A 指向的是该值的结构.
